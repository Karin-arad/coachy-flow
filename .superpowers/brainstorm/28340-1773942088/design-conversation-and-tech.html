Responsive Strategy

1
Mobile First (320px - 430px)
Base styles. Full-width cards, stacked layout, large touch targets. This is the primary experience.
2
Tablet (431px - 768px)
Centered card with max-width 430px. Cream background visible on sides. Same mobile layout inside.
3
Desktop (769px+)
Phone-shaped container centered on screen with subtle shadow. "App in browser" feel - like a phone preview.

iOS / Safari Fixes

  • Replace 100vh with 100dvh (dynamic viewport height) - fixes iOS address bar issues
  • Remove all iOS-specific CSS hacks - clean slate with modern CSS that works everywhere
  • Use touch-action: manipulation - prevents 300ms tap delay on iOS
  • Custom slider with touch events - replace native range input for consistent cross-browser look
  • Keyboard-aware layout - conversation textarea adjusts when iOS keyboard opens
  • Safe area insets - proper padding for notch/home indicator

Animations & Transitions

  • Screen transitions: Smooth slide + fade between steps (keep Framer Motion)
  • Card entrance: Subtle scale-up (0.95 → 1.0) + fade on mount
  • Slider interaction: Haptic-feeling snap with emoji size change based on value
  • Button press: Slight scale-down (0.97) on tap for tactile feel
  • Celebration: Keep confetti but add warm color palette (orange, peach, gold)
  • Loading state: Pulsing orange dots instead of generic spinner

✅ Full Design Summary

Palette: Warm orange + peach + cream • Typography: Rubik with clean weight hierarchy • Layout: Mobile-first, centered app container on larger screens • Components: Dot progress, warm card sliders, grid time picker, rich workout card • Conversation: Chat-like bubbles with quick-pick tags • iOS: Modern CSS fixes, no hacks • Animations: Refined Framer Motion transitions
